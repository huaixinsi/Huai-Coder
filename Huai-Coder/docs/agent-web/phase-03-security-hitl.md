# 阶段三：工具安全与人工审批

## 目标

在开放文件写入和命令执行前，建立独立于 LangGraph 的安全和审批层。

## 核心原则

LangGraph 负责暂停和恢复；业务安全代码负责判断是否允许执行。任何 Agent 框架默认工具都不能绕过本地策略。

## 后端任务

1. 实现自定义 Tool Registry。
2. 增加 `write_file` 和 `execute_command`。
3. 实现工作区路径校验。
4. 实现敏感路径策略。
5. 实现 Bash、PowerShell 和 cmd 基础风险分析。
6. 实现审批策略和审批状态表。
7. 使用 LangGraph interrupt 暂停 Graph。
8. 实现审批恢复、拒绝和取消。
9. 实现审计日志。

## 前端任务

1. 实现审批弹窗。
2. 展示工具名称、参数、风险原因和目标文件。
3. 支持批准、拒绝、取消。
4. 展示审计结果和错误。

## 验收

- 越界路径直接拒绝。
- `.env`、密钥、SSH 和云凭证等敏感路径需要额外审批或禁止访问。
- 高风险命令不能自动执行。
- 审批后可以恢复原 Agent 运行。
- 所有允许和拒绝结果可审计。
